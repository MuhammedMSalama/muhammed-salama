# Security Policy

## Supported Versions

This repository hosts a static portfolio site. Only the latest released version is maintained; security fixes are not backported to older tags.

| Version | Supported |
|---|---|
| 1.1.x | ✅ |
| 1.0.x | ❌ |
| < 1.0 | ❌ |

## Reporting a Vulnerability

If you discover a security issue in this repository — for example, an XSS vector in how page fragments are loaded and rendered, a metadata leak, or a misconfigured header — please report it privately rather than opening a public issue.

**Email:** [devmuhammedsalama@gmail.com](mailto:devmuhammedsalama@gmail.com)

When reporting, please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, including any relevant URLs, payloads, or browser/environment details
- A proof of concept, if available

## Expected Response Timeline

| Stage | Timeframe |
|---|---|
| Initial acknowledgment | Within 48 hours |
| Preliminary assessment | Within 5 business days |
| Fix or mitigation | Depends on severity — critical issues are prioritized immediately |

You will receive updates as the report is triaged and resolved.

## Responsible Disclosure

Please give me a reasonable window to investigate and address a reported issue before disclosing it publicly. In return, I will:

- Acknowledge your report promptly and keep you informed of progress
- Credit you (if you'd like) once the issue is resolved
- Avoid pursuing legal action against good-faith security research conducted under this policy

Please do not:

- Access, modify, or exfiltrate data that isn't yours
- Perform testing that could degrade the availability of the site for other users
- Publicly disclose the issue before a fix has been released
